# AliLocal — Chrome Extension
## Comparador AliExpress vs. Tiendas Locales de Israel

---

## Contexto del proyecto

AliLocal es una extensión de Chrome que aparece automáticamente cuando el usuario
está en una página de producto de AliExpress. Compara el precio del producto con
tiendas locales israelíes (online y físicas), calculando el costo real incluyendo
IVA de importación, tiempo de espera de envío, y distancia a tiendas físicas.

**El problema que resuelve:**
Desde junio 2026, Israel bajó la exención de IVA en importación personal a $75.
Compras entre $75-$500 pagan 18% de IVA en aduana. El usuario no sabe si realmente
le conviene comprar en AliExpress o ir a una tienda local. AliLocal hace ese cálculo
automáticamente.

**Diferenciador único:**
Nadie en el mundo compara AliExpress contra tiendas físicas locales con
geolocalización. Los competidores (AliPrice, Pricearchive, Cheaperly) solo comparan
entre plataformas online globales.

---

## Stack técnico

- **Extensión:** Chrome Manifest V3 (content script + panel lateral)
- **Backend:** FastAPI + Python 3.11 (corre en localhost:8000 para desarrollo)
- **Matching engine:** Claude API (Haiku para extracción, Sonnet para verificación)
- **Búsqueda local:** Google Custom Search API (tiendas online Israel)
- **Tiendas físicas:** Google Places API (geolocalización)
- **Cache:** Supabase (evitar llamadas repetidas para el mismo producto)
- **Afiliados:** AliExpress Affiliate API (el developer ya tiene cuenta registrada)

## Lógica de cálculo de IVA (importante)

```python
USD_TO_ILS = 3.75  # actualizar según tipo de cambio

def calculate_real_cost(price_usd: float) -> dict:
    price_ils = price_usd * USD_TO_ILS
    
    # Reglas actuales Israel (junio 2026):
    # - Hasta $75: sin IVA, sin arancel
    # - $75 a $500: IVA 18% sobre el valor del producto
    # - Más de $500: IVA 18% + arancel aduanero adicional
    
    if price_usd <= 75:
        vat = 0
        vat_applies = False
    elif price_usd <= 500:
        vat = price_ils * 0.18
        vat_applies = True
    else:
        vat = price_ils * 0.18  # más arancel, calcular por categoría
        vat_applies = True
    
    return {
        "price_ils": round(price_ils, 2),
        "vat_applies": vat_applies,
        "vat_amount_ils": round(vat, 2),
        "total_cost_ils": round(price_ils + vat, 2),
        "shipping_days_estimate": 18  # promedio AliExpress a Israel
    }
```

## Extensión Chrome — content.js

El content script debe:

1. **Detectar** que estamos en una página de producto de AliExpress.
   URL pattern: `https://www.aliexpress.com/item/*.html`

2. **Extraer del DOM:**
   - `item_id`: del URL (el número largo antes de .html)
   - `title`: selector `h1[data-pl="product-title"]` o similar
   - `price_usd`: selector de precio principal
   - `specs`: tabla de especificaciones si existe

3. **Obtener geolocalización** del usuario con `navigator.geolocation.getCurrentPosition()`

4. **Llamar al backend:**
   ```javascript
   fetch('http://localhost:8000/match', {
     method: 'POST',
     headers: {'Content-Type': 'application/json'},
     body: JSON.stringify({item_id, title, price_usd, specs, user_lat, user_lng})
   })
   ```

5. **Mostrar el panel lateral** con los resultados.

## Extensión Chrome — Panel lateral (UI)

El panel aparece en el lado derecho de la pantalla, superpuesto sobre AliExpress.
Ancho: 320px. Alto: automático según contenido. Fondo blanco con sombra.

**Estados del panel:**

### Cargando:
- Logo AliLocal pequeño
- Spinner animado
- Texto: "מחפש מחירים מקומיים..." (Buscando precios locales...)

### Resultado encontrado:

### Sin resultado:
- Mensaje: "לא נמצאו חנויות מקומיות" (No se encontraron tiendas locales)
- Botón: "בקש חיפוש ידני" (Solicitar búsqueda manual)
  → Este botón guarda la petición en Supabase para que el operador
    (Martín) la revise y contacte tiendas manualmente.

### Diseño:
- Fuente: sistema (no cargar fuentes externas para velocidad)
- Colores: blanco fondo, #FF4747 acento (similar al rojo de AliExpress),
  verde #22C55E para "más barato local", azul #3B82F6 para links
- Botón cerrar [✕] en esquina superior derecha
- El panel NO debe tapar el botón "Buy Now" de AliExpress

## Flujo semi-automático (operación manual)

Cuando el panel muestra "Sin resultado" y el usuario presiona
"Solicitar búsqueda manual", el backend guarda en Supabase:

```sql
-- Tabla: manual_requests
CREATE TABLE manual_requests (
  id uuid primary key default gen_random_uuid(),
  item_id text,
  title text,
  price_usd numeric,
  user_lat numeric,
  user_lng numeric,
  status text default 'pending',  -- pending / contacted / resolved
  store_name text,
  store_price_ils numeric,
  created_at timestamptz default now()
);
```

El operador (Martín) revisa estas peticiones en un dashboard simple,
llama a la tienda, y actualiza el registro. El usuario recibe
notificación push de Chrome cuando hay resultado.

## Objetivo del MVP (hoy)

1. ✅ Extensión carga en Chrome modo desarrollador sin errores
2. ✅ Al entrar a cualquier producto de AliExpress aparece el panel
3. ✅ El panel muestra datos mockeados correctamente
4. ✅ El cálculo de IVA funciona según el precio del producto
5. ✅ Los botones "Ver en tienda" y "Cómo llego" abren links correctos
6. ✅ El botón [✕] cierra el panel

Lo que NO es objetivo hoy:
- Conectar APIs reales de Google
- Deploy en Hetzner
- Notificaciones push
- Dashboard de peticiones manuales

---

## Notas importantes

- El panel debe aparecer automáticamente, sin que el usuario haga click
- AliExpress cambia sus selectores del DOM frecuentemente — usar múltiples
  selectores en cascada como fallback
- CORS: el backend debe permitir requests desde la extensión
  (`allow_origins=["chrome-extension://*"]` en FastAPI)
- La extensión debe funcionar aunque el backend esté caído (mostrar mensaje
  amigable, no romperse)
- Idioma del panel: hebreo con números en inglés (estándar israelí)
